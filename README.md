# 10分ブログ工場

AI×仕事効率化メディア「AI仕事術ラボ」を、**1日10分**で運営するためのシステム一式。

執筆・SEO・HTML生成・公開準備はClaudeが行い、人間は「確認してOKを出す」だけ。

## あなたの毎日(10分)

1. Claude Code を開いて `/daily` と打つ
2. Claudeが記事を書き、サイトを更新する(待ち時間 約2分)
3. できあがった記事をブラウザでざっと読む(5分)
4. OKなら「公開して」と言う。直したければ指示する

以上。土曜だけ数字の確認が加わります。

## 初回セットアップ(1回だけ、約30分)

### 1. サイトをネットに公開する(無料)

GitHub Pages を使います。

1. [github.com](https://github.com) でアカウント作成(無料)
2. 新しいリポジトリを作る(名前例: `ai-shigoto-lab`、**Public** にする)
3. Claude Code でこう頼む:「GitHubにリポジトリを作ったからpushして公開設定を教えて」
   - Claudeが remote 設定と push を行います
4. GitHubのリポジトリページ → Settings → Pages → Branch を `main` / `docs` フォルダに設定
5. 数分後、`https://あなたのID.github.io/リポジトリ名/` でサイトが公開される
6. 公開URLが決まったら Claude に「config.json の siteUrl を本番URLにして」と頼む

※後で本格運営するなら独自ドメイン(年1,500円程度)推奨ですが、最初は無料でOK。

### 2. 収益化の登録(無料)

[STRATEGY.md](STRATEGY.md) の「今すぐやる登録」を参照。もしもアフィリエイトとA8.netに登録。

### 3. アクセス解析

Google Search Console にサイトを登録(やり方はClaudeに聞けば手順を案内します)。

## ファイル構成

```
config.json    サイト設定(タイトル・URL)
build.js       サイト生成エンジン(node build.js で実行)
posts/         記事の原稿(Markdown)
pages/         固定ページ(運営者情報・プライバシーポリシー)
assets/        デザイン(CSS)
docs/          生成された公開用サイト(自動生成、直接編集しない)
topics.md      記事ネタ帳(50本)と執筆ルール
STRATEGY.md    収益化戦略・ロードマップ・撤退基準
.claude/commands/daily.md   /daily コマンドの定義
```

## よくある操作

| やりたいこと | Claudeへの頼み方 |
| --- | --- |
| 今日の記事を書く | `/daily` |
| サイトを手動で再生成 | 「ビルドして」 |
| 記事を直す | 「○○の記事のここを直して」 |
| 広告リンクを貼る | 「この記事にこのアフィリエイトリンクを入れて」(リンクのコードを貼る) |
| サイト名やデザイン変更 | 「サイト名を○○に変えて」「もっと暖色系のデザインにして」 |
| 戦略の相談 | 「STRATEGY.mdを踏まえて今月の方針を相談したい」 |
