AIに頼んだものの、「なんか違うな…」という答えが返ってくることはよくあります。

そんな時、多くの人は**最初から全部入力し直そう**としますが、それは非効率。AIは会話を覚えているので、**ひとこと言い直すだけ**で軌道修正できます。

この記事では、「ズレた答え」を理想に近づける**5つの直し方**を、そのまま使える追加指示つきで紹介します。

なお、そもそもの頼み方のコツは[AIプロンプトの基本「役割・条件・形式」の3点セット](prompt-kihon-3ten.html)で解説しています。あわせて読むと、やり直しの回数自体を減らせます。

## パターン1:回答が長すぎる・くどい

要点だけ欲しいのに、前置きや説明が長い時。

```
要点だけ、箇条書きで3つにまとめて。
```

```
結論を先に、1行で言うと?
```

## パターン2:内容が浅い・一般論ばかり

「どれも当たり前のことだな」と感じる時は、**深掘り**を要求します。

```
もっと具体的に。実際の例を1つ加えて説明して。
```

```
初心者がやりがちな失敗も含めて、踏み込んで教えて。
```

## パターン3:そもそも的外れ・質問を誤解している

求めているものと違う方向の答えが来た時は、**前提を補足**します。叱る必要はありません。

```
ごめん、伝え方が悪かった。私が知りたいのは〇〇についてです。△△の観点で答えて。
```

ポイントは「何が違うのか」を一言添えること。AIは“何を外したか”が分かれば、すぐ修正できます。

## パターン4:言葉が難しい・専門的すぎる

専門用語が多くて理解しづらい時。

```
中学生にも分かるように、やさしい言葉で言い直して。
```

```
専門用語には、それぞれ一言で説明を添えて。
```

## パターン5:トーン(雰囲気)が合わない

文章の堅さ・柔らかさが用途に合わない時。

```
もっとカジュアルに、親しみやすい感じで書き直して。
```

```
ビジネスメールにふさわしい、丁寧なトーンにして。
```

## やり直しを減らす根本のコツ

これらは「出てきた答えを直す」テクニックですが、そもそも**最初の頼み方**を工夫すれば、やり直しの回数は大きく減ります。

- **役割・条件・形式の3点を最初に伝える**(→ [プロンプトの基本記事](prompt-kihon-3ten.html))
- **長い作業は一度に頼まず、ステップに分けて頼む**
- **お手本を1つ見せる**(「こういう感じで」と例を渡すと精度が上がる)

完璧な指示を一発で出す必要はありません。**「だいたいで頼んで、会話で仕上げる」**のがAIとの一番ラクな付き合い方です。

## まとめ

- 答えがズレても入力し直さず、ひとことで軌道修正する
- 「長い→要点だけ」「浅い→具体例を」「的外れ→前提を補足」など状況別に言い直す
- やり直しを減らすには、最初の頼み方(3点セット)が効く

AIは何度でも付き合ってくれます。遠慮なく「もう一回」「こう直して」と頼んで、理想の答えにたどり着いてください。
